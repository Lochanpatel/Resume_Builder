Overview

Goal: Build Resume_Builder incrementally and deliver a working MVP within 3–4 sprints.

Cadence

- Sprint length: 2 weeks (adjustable)
- Sprint ceremonies: Planning, Refinement, Review, Retrospective


Definition of Done (DoD)

- Code merged via PR and reviewed
- Unit / integration tests added and passing
- Linting and CI checks pass
- Basic documentation / usage note updated
- Deployed to staging 

Quality & Metrics

- Automated tests (unit + integration) for core modules
- Target >80% coverage for core logic (MVP)
- Track velocity (story points/sprint), cycle time, and escaped defects

Tools

- Issues & backlog: GitHub Issues / Projects
- CI: GitHub Actions (recommended)
- Communication: Slack / Teams
- Design: Figma (for templates)
